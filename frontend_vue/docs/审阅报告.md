# frontend_vue 审阅报告

## 1. 审阅范围
- 目录：`frontend_vue/src`
- 关注点：页面路由、鉴权状态、API 对接、交互一致性、错误处理、可维护性

## 2. 总体结论
- 结论：**可联调，未达生产级**。
- 现状：已完成路由化、鉴权拆分、社区状态拆分、推荐与搜索接口接入。
- 风险：错误处理与可观测性仍偏弱，接口分页/缓存策略不足。

## 3. 分级问题清单

### P0（阻断项）
1. 后端异常时交互提示粒度不足（统一为通用失败文案）。
   - 影响：用户难以判断是登录失效、权限不足还是网络问题。
   - 建议：按 `401/403/5xx` 分类展示并落日志。

2. 会话依赖 Cookie，跨域策略若变化会导致登录态丢失。
   - 影响：环境切换后容易“能登录但请求未授权”。
   - 建议：统一环境配置模板，前端启动时做连通性自检。

### P1（高优先）
1. 列表与评论缺少分页参数。
   - 影响：数据规模增长后性能下降。
   - 建议：在 API 层统一支持 `page/pageSize` 或 `cursor/limit`。

2. 状态派生链较长，仍有维护成本。
   - 影响：后续新增互动类型时容易漏同步。
   - 建议：进一步收敛为“后端主源 + 前端最小派生”。

### P2（优化项）
1. 缺少端到端自动化验证脚本。
2. 缺少统一埋点与性能指标看板。

## 4. 已完成的关键改进
- 请求层支持超时与统一错误语义。
- 启动阶段会话刷新（`/users/me`）已接入。
- 推荐导航可走后端推荐接口，失败回退热门内容。

## 5. 后续建议迭代
1. 增加分页与缓存策略。
2. 增加 API Mock/Real 一体化契约测试。
3. 增加 E2E 用例：登录、发布、编辑、互动、推荐。
