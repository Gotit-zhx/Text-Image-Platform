# recommend_system 使用说明

## 1. 目录准备
- 项目根目录：`/vifile/tip/recommend_system`
- 数据文件：`data/daily_interactions.csv`

## 2. 训练方式
可由后端统一触发（推荐）：
```bash
cd /vifile/tip/backend_django
/home/zhx/miniconda3/envs/tip/bin/python manage.py train_recsys --trials 10 --val-ratio 0.1 --min-rows 10
```

也可直接在推荐目录调用（需按脚本参数适配）：
```bash
cd /vifile/tip/recommend_system
python recommender.py --mode train --input data/daily_interactions.csv
```

## 3. 常用调优项
- `trials`：超参搜索轮数
- `val-ratio`：验证集比例
- `min-rows`：最小样本行数阈值

## 4. 故障排查
- 数据过少：降低 `min-rows` 或补充交互
- 训练报错：检查 CSV 列完整性与空值
- 推理无结果：确认 `trained_model.npz` 是否生成

## 5. 建议流程
1. 每日凌晨导出最新交互
2. 定时训练并记录状态
3. 线上加载新模型并观察推荐点击率

## 6. 自动化检查与排查
推荐先做导出链路检查：
```bash
cd /vifile/tip/backend_django
/home/zhx/miniconda3/envs/tip/bin/python manage.py train_recsys --export-only
```

再做完整训练：
```bash
/home/zhx/miniconda3/envs/tip/bin/python manage.py train_recsys --trials 10 --val-ratio 0.1 --min-rows 10
```

排查顺序：
1. `--export-only` 是否成功生成 `daily_interactions.csv`
2. 训练状态文件 `recsys_train_status.json` 的 `status/error`
3. 模型文件 `trained_model.npz` 是否存在且更新时间正确
