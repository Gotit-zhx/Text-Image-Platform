# backend_django 使用说明

## 1. 环境
- Python 3.11
- conda 环境：`tip`

## 2. 安装与启动
```bash
cd /vifile/tip/backend_django
conda activate tip
/home/zhx/miniconda3/envs/tip/bin/python -m pip install -r requirements.txt
/home/zhx/miniconda3/envs/tip/bin/python manage.py migrate
/home/zhx/miniconda3/envs/tip/bin/python manage.py seed_demo_data
/home/zhx/miniconda3/envs/tip/bin/python manage.py runserver 127.0.0.1:8000
```

## 3. 推荐训练命令
```bash
/home/zhx/miniconda3/envs/tip/bin/python manage.py train_recsys --trials 10 --val-ratio 0.1 --min-rows 10
```

## 4. 训练状态查看
- 文件：`/vifile/tip/backend_django/recsys_train_status.json`
- 接口：`GET /api/community/recommend/status`

## 5. 每日定时训练（示例）
```cron
0 2 * * * cd /vifile/tip/backend_django && /home/zhx/miniconda3/envs/tip/bin/python manage.py train_recsys --trials 20 --val-ratio 0.1 --min-rows 50 >> /vifile/tip/backend_django/recsys_train.log 2>&1
```

## 6. 常见问题
- 报 `未登录`：检查前端是否携带 Cookie
- 推荐无结果：先执行 `seed_demo_data` 与 `train_recsys`
- 训练失败：查看 `recsys_train_status.json` 的 `error` 字段

## 7. 自动化测试与排查
运行后端自动化测试：
```bash
cd /vifile/tip/backend_django
/home/zhx/miniconda3/envs/tip/bin/python manage.py test
```

当前已覆盖用例：
- 社区初始化接口可用性
- 发帖鉴权拦截（未登录返回 401）
- 注册后发帖成功流程
- 仅作者可编辑权限校验
- 点赞/评论/删评联动流程
- 禁止关注自己

若失败，优先排查：
1. 数据库迁移是否完成（`manage.py migrate`）
2. 依赖是否完整安装（`pip install -r requirements.txt`）
3. 最近改动是否改变响应结构（`code/message/data/requestId`）
