# 项目文档总览（论文参考）

## 1. 文档组织结构

### 1.1 总体文档
- `系统详细文档.md`：全栈架构、数据流、接口、训练、验证与论文写作要点。

### 1.2 前端文档
- `frontend_vue/docs/功能说明.md`
- `frontend_vue/docs/使用说明.md`
- `frontend_vue/docs/审阅报告.md`

### 1.3 后端文档
- `backend_django/docs/功能说明.md`
- `backend_django/docs/使用说明.md`
- `backend_django/docs/审阅报告.md`

### 1.4 推荐系统文档
- `recommend_system/docs/功能说明.md`
- `recommend_system/docs/使用说明.md`
- `recommend_system/docs/审阅报告.md`

---

## 2. 论文写作建议引用顺序
1. 先读 `系统详细文档.md`（用于“系统设计与实现”章节）。
2. 再读三端“功能说明”（用于模块设计章节）。
3. 读“使用说明”（用于实验环境与复现实验章节）。
4. 读“审阅报告”（用于局限性与改进方向章节）。

---

## 3. 文档维护规范
1. 业务接口变化后，先更新对应模块“功能说明”。
2. 启动/训练命令变化后，更新“使用说明”。
3. 阶段性验收后，更新“审阅报告”中的问题分级与结论。
4. 论文终稿前，对齐所有文档中的版本号与接口清单。

---

## 4. 一键运行脚本
- 脚本：`run_project.sh`
- 用法：
  - `./run_project.sh`
  - `./run_project.sh --install`
