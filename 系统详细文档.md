# 图文分享平台（Vue + Django + ALS）系统文档（论文参考版）

## 1. 课题定位与目标
本项目面向“社区图文内容分发 + 个性化推荐”场景，完成了从业务功能到推荐闭环的完整工程实现，目标包括：

1. 构建可用的图文社区系统（发布、评论、关注、收藏、审核）。
2. 采集用户行为并形成推荐训练数据（`read/like/favorite/share`）。
3. 采用 ALS 进行隐式反馈推荐，支持离线训练、在线推理、失败回退。
4. 提供可复现、可联调、可验证的工程文档与测试路径。

---

## 2. 系统总体架构

### 2.1 架构分层
- 前端层（`frontend_vue`）：Vue3 + TS + Element Plus，负责 UI 与交互。
- 业务服务层（`backend_django`）：认证、社区、后台管理、审计。
- 推荐计算层（`recommend_system`）：行为矩阵、模型训练、推荐生成。

### 2.2 数据流闭环
1. 用户在前端产生行为（浏览、点赞、收藏、分享等）。
2. 后端将行为落库为 `PostInteraction`。
3. `train_recsys` 导出交互 CSV 并训练 ALS 模型。
4. 在线推荐接口加载模型，返回个性化结果。
5. 模型不可用时回退热门内容，保证服务可用性。

## 3. 目录与模块职责

### 3.1 前端关键目录
- `frontend_vue/src/api`：接口调用封装。
- `frontend_vue/src/composables`：鉴权与社区状态管理。
- `frontend_vue/src/components`：首页、详情、发布、个人中心、后台组件。

### 3.2 后端关键目录
- `backend_django/community/models.py`：业务实体定义。
- `backend_django/community/views.py`：REST API 与管理接口。
- `backend_django/community/recsys_pipeline.py`：导出与训练触发。
- `backend_django/community/recommendation.py`：模型加载与推荐回退。

### 3.3 推荐系统目录
- `recommend_system/recsys/data.py`：交互数据处理与映射。
- `recommend_system/recsys/trainer.py`：ALS 训练与评估。
- `recommend_system/recommender.py`：训练入口脚本。

## 4. 运行环境与启动

### 4.1 环境要求
- Python 3.11（建议 conda 环境 `tip`）
- Node.js 18+
- 后端解释器：`python3` 或 `{{your-python-path}}`

### 4.2 后端启动
```bash
cd backend_django
# 进入项目顶层路径（git clone 后）：
# cd /your/path/to/tip/backend_django

# 选择你自己的 Python 解释器
python3 -m pip install -r requirements.txt
python3 manage.py migrate
python3 manage.py seed_demo_data
python3 manage.py runserver 127.0.0.1:8000
```

### 4.3 前端启动
前端 `.env`：
```env
VITE_API_BASE_URL=http://127.0.0.1:8000/api
VITE_USE_MOCK_API=false
```

```bash
cd frontend_vue
npm install
npm run dev
```

## 5. 推荐训练与产物

### 5.1 一次训练命令
```bash
cd backend_django
python3 manage.py train_recsys --trials 10 --val-ratio 0.1 --min-rows 10
```

### 5.2 产物文件
- 导出数据：`recommend_system/cache/daily_interactions.csv`
- 模型文件：`recommend_system/cache/trained_model.npz`
- 状态文件：`backend_django/recsys_train_status.json`

### 5.3 定时任务示例
```cron
0 2 * * * cd backend_django && python3 manage.py train_recsys --trials 20 --val-ratio 0.1 --min-rows 50 >> backend_django/recsys_train.log 2>&1
```

## 8. 可用性与安全策略
1. 未登录访问受限接口统一返回 401 JSON。
2. 前端默认携带 Cookie，适配 Django Session。
3. 推荐接口模型异常时自动回退热门。
4. 小样本训练保护（`min-rows`）避免无效训练。
5. 富文本渲染已接入净化策略（DOMPurify）。
